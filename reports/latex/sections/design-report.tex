\section{设计报告}

\subsection{需求分析}

本实验要求基于 RINEX 观测文件与广播星历文件，完成北斗/GNSS 定位解算全流程软件系统。系统需要实现 RINEX 数据导入、观测数据预处理、卫星位置与钟差计算、传播延迟修正、伪距单点定位、连续定位分析、结果可视化和 GUI 交互。

结合实验文档中 5 个核心模块，项目的主要需求如下：

\begin{enumerate}
\item 支持 RINEX 观测文件与导航文件解析，提取卫星编号、伪距、信噪比、广播星历参数和测站近似坐标。
\item 基于广播星历手写实现 GPS/BDS 卫星位置计算、钟差修正、相对论效应修正、对流层与电离层延迟改正。
\item 构建伪距定位观测方程，采用迭代最小二乘解算接收机 ECEF 坐标、经纬高坐标和接收机钟差。
\item 对多历元数据进行连续定位，输出定位轨迹、误差统计、DOP 和卫星可见数变化。
\item 提供命令行与 PyQt GUI 两类入口，实现数据导入、参数设置、实时显示、轨迹回放和误差图查看。
\end{enumerate}

核心算法均由项目代码手写实现，没有调用 RTKLIB、georinex、pymap3d 等第三方定位库。第三方依赖主要用于 GUI、绘图和基础运行环境。

\subsection{数据结构设计}

项目主要数据结构定义在 \texttt{src/models.py} 中：

\begin{itemize}
\item \texttt{ObsHeader}：保存观测文件版本、测站名、近似坐标、观测类型和时间系统。
\item \texttt{ObsEpoch}：保存单历元时间、历元标志和以卫星编号为键的观测值字典。
\item \texttt{NavRecord}：保存单颗卫星的一组广播星历参数，包括轨道参数、钟差参数、TGD 和健康状态。
\item \texttt{NavHeader}：保存导航文件头信息，包括 GPS/BDS 电离层参数和闰秒信息。
\item \texttt{PositionSolution}：保存定位结果、接收机钟差、使用卫星数、DOP 和残差诊断指标。
\end{itemize}

这些结构将 RINEX 文件、卫星计算和定位结果解耦，使各模块之间通过明确的数据对象传递信息，便于调试和后续扩展。
