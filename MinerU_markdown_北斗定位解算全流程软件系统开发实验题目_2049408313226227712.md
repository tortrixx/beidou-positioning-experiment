# 中级项目课 （电科专业）

# 北斗定位解算全流程软件系统开发实验

# 一、实验目的

1. 以软件编程为纽带，深度融合电子科学与技术核心理论，将电路设计、信号处理等专业知识转化为可编程实践，掌握北斗卫星导航定位核心原理。

2. 熟练掌握 RINEX 格式数据解析、卫星位置与钟差计算、伪距单点定位解算、连续定位与精度分析的核心方法，能独立完成全流程软件开发。

3. 培养模块化编程、代码规范、系统调试能力，模拟真实开发场景，提升工程实践与自主探究能力。

4. 为后续高级项目课奠定软件算法与工程化基础。

# 二、实验内容（60分）

本实验为纯软件开发实验，基于 RINEX 观测数据与广播星历数据，完成北斗定位解算全流程软件系统开发，分为5 个核心模块，按以下步骤完成：

# 模块1：RINEX数据解析

1. 解析 RINEX 观测文件（*.obs）与导航文件（*.nav），提取卫星编号、伪距观测值、信噪比、广播星历参数等核心数据。

2. 完成数据预处理：剔除不健康卫星、无效观测值，进行伪距粗差初步剔除，按高度角筛选可见卫星，实现同一历元多卫星数据时间对齐。

# 模块2：卫星位置与钟差计算

1. 基于广播星历数据，计算北斗卫星在轨位置（ECEF 坐标系），包含轨道摄动修正，输出卫星X/Y/Z坐标。

2. 完成卫星钟差修正（多项式修正 $^ +$ 相对论效应修正），修正伪距观测值。

3. 实现传播延迟修正：采用 Saastamoinen 模型完成对流层延迟修正，采用简化模型完成电离层延迟修正，输出最终修正后的伪距。

# 模块3：单点定位解算核心算法

1. 筛选历元内可见卫星集合，计算 PDOP、GDOP 等几何精度因子，输出可用卫

星数量与几何精度。

2. 构建伪距定位观测方程，采用迭代最小二乘算法求解用户位置（ECEF 坐标系）与用户钟差，设置合理的迭代收敛判断条件。

3. 完成 ECEF 坐标系与经纬高（BLH）坐标系转换，输出标准定位结果（经度、纬度、高程）。

# 模块4：连续定位与结果分析

1. 实现逐历元循环解算，完成多历元连续定位，输出用户连续定位轨迹，存储定位结果时间序列。

2. 进行定位精度评估：计算定位结果的 RMS 误差、均值误差、最大误差，分析卫星数量、DOP 值与定位精度的关系。

3. 完成结果可视化：绘制定位误差曲线、经纬度轨迹图、卫星可见数与 DOP值变化曲线（至少完成1种可视化图表）。

# 模块5：软件系统整合与测试

1. 采用模块化架构设计，将上述 4 个模块整合为完整软件系统，实现“数据输入 $\twoheadrightarrow$ 预处理 $\twoheadrightarrow$ 解算 $\twoheadrightarrow$ 分析 $\twoheadrightarrow$ 输出”全流程自动化。

2. 开发简单可视化交互界面（使用 QT、PyQt等工具），实现RINEX数据导入、解算参数（迭代次数、误差阈值）设置、定位结果实时显示、定位轨迹回放及误差曲线查看功能，完善软件交互体验，确保界面操作简洁、功能可用。

3. 使用多组 RINEX 数据测试软件功能，记录测试结果，排查系统漏洞，输出测试报告。

# 三、实验要求

1. 编程要求：自由使用 $\mathrm { C } / \mathrm { C } { + } +$ 或 Python/Java 等语言编程 ，核心算法（卫星位置解算、最小二乘解算等）需手写实现，禁止直接调用第三方定位库。

2. 功能要求：完成上述 5 个模块的全部核心功能，软件能正常运行，全流程无明显漏洞，能输出正确的定位结果与精度分析报告。

3. 代码要求：代码命名规范、结构清晰，注释完整（关键函数、核心算法需详细注释），无冗余代码、语法错误，符合工程编程规范。

4. 文档要求：提交完整的实验报告（含设计报告与实验报告），设计报告需包含需求分析、功能设计（架构图、流程图）；实验报告需包含原理阐述、模块实现过程、测试结果、误差分析、问题与改进思路。

5. 提交要求：按时提交软件源码（含所有模块代码）、实验报告、测试数据与结果截图；严禁抄袭代码、报告。

6. 分层要求：基础薄弱同学可优先完成各模块基础功能，学有余力同学可在基础功能之上，拓展优化（见附加题）。

# 四、附加题（可选，10 分）

说明：附加题为可选任务，欢迎学有余力的同学选做，完成后可在总分基础上加分（最高加 10 分）；不做附加题，实验最高得分为 60 分，完成附加题且合格，最高可获得70分。

结合AI技术，实现基于简单机器学习模型（如线性回归、随机森林）的定位误差预测与补偿，具体要求如下：

1. 数据准备：利用实验中生成的多组连续定位数据（至少 3组不同场景的RINEX 数据解算结果）作为历史数据，提取定位误差相关特征（如卫星数量、DOP值、伪距观测值、延迟修正量等），划分训练集（ $7 0 \%$ ）与测试集（ $3 0 \%$ ），规范数据格式以适配模型训练；

2. 模型构建与训练：选择线性回归或随机森林模型（二选一即可，鼓励尝试两种模型对比），使用 Python 相关库（如 scikit-learn）实现模型构建，设置合理的模型参数，利用训练集完成模型训练，调试参数优化模型拟合效果；

3. 误差补偿实现：将训练好的模型集成到原有北斗定位软件中，实现定位误差实时预测与自动补偿功能，即软件解算得到初始定位结果后，通过模型预测该结果的误差值，并自动对定位结果进行补偿修正；

4. 效果验证与分析：使用测试集数据验证模型性能，对比补偿前后的定位精度指标（RMS误差、均值误差、最大误差），绘制精度对比曲线；

5. 撰写完整技术实现报告，明确模型选择依据、数据处理过程、模型训练细节、补偿实现步骤，分析模型在北斗定位误差补偿中的优势、不足及改进方向，体现AI技术与北斗定位软件开发的融合思路。
